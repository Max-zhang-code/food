# 菜品食材管理 Design

> Stage 1 | 2026-07-02（修订）

## 0. 术语约定

无新增术语。沿用 `CONTEXT.md` 已有定义。

## 1. 决策与约束

### 需求摘要

- **做什么**：审核人在统一管理页中查看/搜索/审核/编辑/新增菜品和食材。管理页合并了原来分离的"审核"和"提交"两个入口，加搜索框和状态筛选。
- **为谁**：仅审核人。
- **成功标准**：审核人从「我的」进入管理页，顶部搜索+状态筛选，卡片列表含审核操作（通过/驳回）和编辑按钮，点新增跳转提交页。
- **明确不做**：删除菜品/食材；非审核人不可见管理入口。

### 复杂度档位

走默认档位。

### 关键决策

| 决策 | 选择 | 原因 |
|---|---|---|
| D1 | 管理页合并审核功能——卡片内直接通过/驳回，替代独立 review 页 | 审核人一屏完成所有操作，减少页面跳转 |
| D2 | 移除「✅ 审核菜品」入口，只留「📋 管理菜品」「📋 管理食材」 | 避免两条入口指向重叠功能 |
| D3 | 搜索框 + 状态筛选 tabs（全部/待审核/已上架/已驳回） | 菜品多时需要快速定位 |
| D4 | 食材管理页也加搜索框（无需状态筛选） | 结构一致 |
| D5 | 编辑复用现有提交页面，审核人可编辑任意菜品 | 已有编辑模式骨架 |

### Top 3 风险

| 风险 | 缓解 |
|---|---|
| R1：审核操作误触 | 通过/驳回前加确认弹窗 |
| R2：搜索+筛选+审核全堆一页，卡片列表性能 | 前端过滤，食材几十条无压力 |
| R3：旧 review 页残留引用 | 移除 mine 入口但保留页面文件（不删路由，避免 404） |

### 关键假设

- 假设：食材数量 < 100，前端搜索过滤无性能问题

## 2. 名词与编排

### 2.1 名词层

**现状**：
- `dishes` 集合，菜品状态机 `pending → approved/rejected`。提交页有编辑模式（仅 rejected 的菜可编辑重提）。
- `ingredients` 集合，无编辑/删除能力。

**变化**：

#### 新增云函数

**`updateDish`** — 审核人编辑任意菜品

```
输入: { dish_id, name?, description?, cooking_time?, image?, ingredient_ids? }
输出: { success: true, dish: {...} }
错误:
  { code: "NOT_APPROVER" }
  { code: "DISH_NOT_FOUND" }
  { code: "DUPLICATE_NAME", message: "已存在同名菜品" }
```

**`updateIngredient`** — 审核人编辑任意食材

```
输入: { ingredient_id, name?, category?, icon?, image? }
输出: { success: true, ingredient: {...} }
错误:
  { code: "NOT_APPROVER" }
  { code: "INGREDIENT_NOT_FOUND" }
  { code: "DUPLICATE_NAME", message: "已存在同名食材" }
```

#### 新增云函数

**`getAllDishes`** — 审核人拉全部菜品（不限状态）

```
输入: {}
输出: { dishes: [...] }
错误: { code: "NOT_APPROVER" }
```

**`getAllIngredients`** — 审核人拉全部食材

```
输入: {}
输出: { ingredients: [...] }
错误: { code: "NOT_APPROVER" }
```

#### 新增页面

- `pages/manage/manage-dishes/index` — 菜品管理列表
- `pages/manage/manage-ingredients/index` — 食材管理列表

#### submit-ingredient 页面扩展编辑模式

通过 URL 参数 `ingredientId` 进入编辑模式，回填表单数据，提交时调 `updateIngredient`。

### 2.2 编排层

#### 管理菜品页布局

```
┌─ 🔍 搜索菜品... ─────────────────┐
├─ [全部] [待审核(3)] [已上架] [已驳回] ─┤
├────────────────────────────────────┤
│  [+ 新增菜品]                      │
│ ┌───────────────────────────┐     │
│ │ [图] 番茄炒蛋    已上架    │     │
│ │ 提交人：张三         [✎]  │     │
│ └───────────────────────────┘     │
│ ┌───────────────────────────┐     │
│ │ [图] 红烧肉      待审核    │     │
│ │ 提交人：李四    [✓通过][✗驳回] │     │
│ └───────────────────────────┘     │
│ ┌───────────────────────────┐     │
│ │ [图] 凉拌黄瓜    已驳回    │     │
│ │ 提交人：王五         [✎]  │     │
│ └───────────────────────────┘     │
└────────────────────────────────────┘
```

- 搜索框：按菜品名实时过滤
- 状态 tabs：切换筛选，括号显示数量
- 待审核卡片：显示「通过」「驳回」按钮（确认弹窗）
- 非待审核卡片：显示当前状态标签 + 编辑按钮
- 前端过滤，不额外调云函数

#### 管理食材页布局

```
┌─ 🔍 搜索食材... ───────────────────┐
│ [全部] [蔬菜] [肉类] [蛋类] [调味] [其他] │  ← 分类筛选
├─────────────────────────────────────┤
│  [+ 新增食材]                       │
│ ┌──────────────┐ ┌──────────────┐  │
│ │ [图] 鸡蛋     │ │ [图] 猪肉     │  │
│ │  蛋类    [✎] │ │  肉类    [✎] │  │
│ └──────────────┘ └──────────────┘  │
└─────────────────────────────────────┘
```

- 搜索框按名称过滤，分类 tabs 按类别筛选
- 卡片网格 2 列，显示图片 + 名称 + 分类标签 + 编辑按钮
- 搜索和分类筛选联动（同时生效）

#### 流程级约束

- **审核人校验**：入口前端 v-if + 云函数双重校验
- **审核确认弹窗**：点通过/驳回时 `uni.showModal` 确认
- **审核后即时更新**：通过/驳回后直接从列表移除或就地更新卡片状态，不重新拉全量
- **编辑后刷新**：navigateBack 后 onShow 重新拉数据
- **菜品编辑不改状态**：updateDish 不修改 status/submitted_by
- **旧 review 页**：保留路由文件但移除 mine 入口

### 2.3 挂载点清单

| # | 挂载位置 | 动作 | 说明 |
|---|---|---|---|
| 1 | `pages.json` manage 分包 | 新增 manage-dishes + manage-ingredients 路由 | 两个新页面 |
| 2 | `pages/mine/index` | 替换「✅ 审核菜品」为「📋 管理菜品」「📋 管理食材」，仅审核人 | 合并入口 |
| 3 | 云函数 | 新增 updateDish, updateIngredient, getAllDishes, getAllIngredients | 4 个云函数 |
| 4 | `pages/submit-ingredient/index` | 扩展编辑模式（ingredientId 参数） | 复用表单 |
| 5 | `pages/submit/index` | 编辑模式从仅 rejected → 审核人可编辑全部 | 扩展现有逻辑 |

5 条。

### 2.4 推进策略

```
1. 云函数（updateDish + updateIngredient + getAllDishes + getAllIngredients）
   退出信号：4 个云函数逻辑完整，含审核人校验

2. 管理列表页（manage-dishes + manage-ingredients）
   退出信号：npm run build 通过，列表展示 + 新增按钮可点

3. submit 页面编辑模式扩展（dish + ingredient）
   退出信号：编辑模式回填数据正常，保存调用正确云函数

4. mine 入口 + 整体联调
   退出信号：审核人可见入口、点编辑→保存→列表刷新全流程通
```

### 2.5 结构健康度与微重构

**文件级**：
- `src/pages/submit/index.vue` — 编辑模式扩展，改动量中等。健康，不做微重构。
- `src/pages/submit-ingredient/index.vue` — 新增编辑模式，需加 onMounted 回填。改动量小。
- `src/pages/mine/index.vue` — 加两条菜单项。改动量小。

**目录级**：
- `src/pages/manage/` — 已有 review/ 和 approvers/，加 manage-dishes/ 和 manage-ingredients/。结构一致。

**结论**：本次不做微重构。
